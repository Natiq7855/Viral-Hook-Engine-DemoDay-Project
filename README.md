**Viral Hook Engine**

Viral Hook Engine is a small TypeScript + React project scaffolded with Vite that demonstrates a frontend app and a minimal server entrypoint. It provides a compact starting point for building interactive UI features backed by a lightweight development server.

**Quick Start**

- **Prerequisites:** Node.js >= 16 and npm or Yarn installed.
- **Install:**

```bash
npm install
# or
yarn
```

- **Run (development):**

```bash
npm run dev
# or
yarn dev
```

- **Build:**

```bash
npm run build
# or
yarn build
```

- **Preview production build:**

```bash
npm run preview
# or
yarn preview
```

**Project Structure**

- **Files:**
- [package.json](package.json) — project metadata and scripts.
- [tsconfig.json](tsconfig.json) — TypeScript configuration.
- [vite.config.ts](vite.config.ts) — Vite build configuration.
- [server.ts](server.ts) — minimal server entry (if present/used).
- [index.html](index.html) — app HTML shell.

- **Source:**
- [src/main.tsx](src/main.tsx) — application bootstrap.
- [src/App.tsx](src/App.tsx) — root React component.
- [src/index.css](src/index.css) — global styles.
- [src/types.ts](src/types.ts) — shared TypeScript types.

**Development Notes**

- The app uses Vite for fast development and HMR. Use `npm run dev` to start a dev server and open the URL printed in the terminal.
- If `server.ts` is used to provide additional backend behavior, start it separately (for example with `ts-node` or by building and running the compiled output) or integrate it into your preferred Node.js/TS runtime.

**Scripts**

- The following scripts are available in `package.json`:
- `dev` — run Vite dev server.
- `build` — produce a production build.
- `preview` — locally preview the production build.

The Problem:
Most productivity gurus tell you to block out 90 minutes of "Deep Work." But in 2026, with the sheer volume of digital pings and the reality of "Context Switching," a 90-minute block feels impossible. Most people spend the first 20 minutes just trying to focus and the last 20 minutes fighting the urge to check their phone.

The Solution: Micro-Sprints
A Micro-Sprint is a 22-minute burst of hyper-focused activity followed by a mandatory 3-minute physical reset.

Why 22 minutes?
It’s short enough to "game" your brain’s resistance to starting, but long enough to enter a flow state. It’s based on the Zeigarnik Effect—the brain’s tendency to remember interrupted tasks better than completed ones. By stopping while you still have momentum, you're actually more eager to start the next sprint.

The 3-Minute Reset Rules:

No Screens: Checking your phone resets the "boredom threshold" and kills your momentum.

Movement: Do 10 air squats or pace the room. Get blood to the prefrontal cortex.

Hydration: Drink 8oz of water. Dehydration is often mistaken for "brain fog."

The Result:
People using Micro-Sprints report a 40% increase in output and, more importantly, they aren't "fried" by 3:00 PM. The goal isn't to work longer; it's to work with higher intensity in shorter bursts.
